Установка Astra Linux
1. Какое минимальное требование к оперативной памяти для установки графической версии Astra Linux Special Edition (релизы последних лет)?
a) 512 МБ
b) 1 ГБ
c) 2 ГБ
d) 4 ГБ
Правильный ответ: c

2. Какой тип разметки диска рекомендуется использовать на современных компьютерах с UEFI для установки Astra Linux?
a) MBR (msdos) с разделом swap и ext4 под корень
b) GPT с разделом EFI (FAT32), разделом под /boot (ext4) и корневым разделом
c) GPT с разделом EFI (FAT32) и корневым разделом (ext4/Btrfs)
d) LVM без отдельного загрузочного раздела
Правильный ответ: c

3. Какой пакетный менеджер используется в Astra Linux для установки обновлений безопасности и исправлений ошибок?
a) yum
b) pacman
c) apt (apt-get, apt)
d) zypper
Правильный ответ: c

4. Что такое "миграция" в контексте документации Astra Linux?
a) Перенос пользовательских данных на новый жесткий диск
b) Чистая установка следующей версии ОС с форматированием разделов
c) Обновление системы на очередной мажорный выпуск (например, с 1.6 на 1.7) с сохранением данных и настроек
d) Установка обновлений ядра без перезагрузки
Правильный ответ: c

5. Во время загрузки системы, какой компонент загружается первым после прохождения аппаратной инициализации (POST)?
a) Ядро Linux (vmlinuz)
b) Загрузчик (GRUB 2)
c) Менеджер дисплеев (LightDM/GDM)
d) Процесс systemd (с PID 1)
Правильный ответ: b

6. Для создания загрузочного USB-накопителя с дистрибутивом Astra Linux в ОС Windows рекомендуется использовать:
a) Стандартное средство форматирования Windows
b) Утилиту Rufus, выбрав режим записи "DD" или "ISO-образ"
c) Программу WinRAR для распаковки ISO на флешку
d) Команду dd в командной строке Windows
Правильный ответ: b

7. Какая последовательность этапов загрузки ОС после инициализации ядра является правильной?
a) Initramfs -> systemd -> монтирование корневой ФС -> запуск целевых (target) юнитов
b) systemd -> Initramfs -> монтирование корневой ФС -> GRUB
c) GRUB -> монтирование корневой ФС -> Initramfs -> systemd
d) Initramfs -> монтирование корневой ФС -> systemd -> запуск целевых юнитов
Правильный ответ: d

8. Перед миграцией на очередной мажорный релиз Astra Linux ОБЯЗАТЕЛЬНО необходимо:
a) Обновить все пакеты до последних версий в текущем релизе
b) Удалить все сторонние репозитории (репозитории не из дистрибутива)
c) Создать полную резервную копию данных и конфигураций системы
d) Все перечисленные действия крайне рекомендуются
Правильный ответ: d

9. Где размещается конфигурация загрузчика GRUB 2 в Astra Linux при установке на диск с UEFI?
a) /boot/grub/grub.cfg
b) /etc/default/grub
c) EFI/astra/grub.cfg на разделе EFI
d) В MBR первого диска
Правильный ответ: c

10. Какая команда обновит списки доступных пакетов из всех подключенных репозиториев?
a) apt upgrade
b) apt update
c) apt dist-upgrade
d) apt install update
Правильный ответ: b

11. Какой раздел НЕ является обязательным для установки Astra Linux в режиме UEFI?

c) Раздел подкачки (swap)

c) Для создания резервной копии списка установленных пакетов

13. Что делает команда update-initramfs -u -k all?

14. Какой параметр загрузки ядра в меню GRUB нужно использовать для устранения неполадок, если система не загружается из-за проблем с графическим драйвером?
a) single

a) apt upgrade -> apt update -> перезагрузка

a) Раздел будет автоматически смонтирован при загрузке

22. При использовании LVM во время установки, какой порядок действий правильный?
a) Создать Physical Volume -> Создать Volume Group -> Создать Logical Volume -> отформатировать его
b) Создать Volume Group -> Создать Physical Volume -> отформатировать диск -> Создать Logical Volume
c) Отформатировать диск -> Создать Logical Volume -> Создать Volume Group -> Создать Physical Volume
d) Создать раздел типа LVM -> Создать на нем файловую систему -> добавить в Volume Group

23. Файл конфигурации /etc/default/grub используется для:
a) Непосредственной загрузки ОС
b) Настройки параметров, на основе которых генерируется основной файл grub.cfg
c) Хранения паролей для загрузки
d) Выбора ядра по умолчанию вручную
Правильный ответ: b

24. Какая команда позволяет установить все отложенные обновления безопасности?
a) apt safe-upgrade
b) apt full-upgrade
c) apt upgrade --security
d) unattended-upgrade
Правильный ответ: d (Правильнее: запуск утилиты unattended-upgrade. В вариантах apt upgrade тоже установит обновления безопасности, но команда unattended-upgrade предназначена именно для автоматической установки критических обновлений).
